设置中心

维护 FNS 同步目标、图片策略和账号安全。保存后立即生效。

FNS 配置

填写 Fast Note Sync 服务信息。导入只填表，需要手动保存。

图片外链设置

选择文章中图片的处理模式。

Telegram Bot 图链

可单独覆盖 Bot 入口的图片处理模式，留空则跟随全局设置。

飞书 Bot 图链

可单独覆盖 Bot 入口的图片处理模式，留空则跟随全局设置。

Telegram Bot

把公众号链接或普通网页链接直接发给 Bot，即可触发单篇转换。Webhook 保存时会自动注册。

飞书 Bot

通过飞书自建应用 Bot 的事件订阅触发单篇转换。第一版仅支持私聊；open_id 白名单可留空，联调完成后再补。

App ID / App Secret 不能自动推导用户 open_id。可先留空，私聊 Bot 发送一条链接后，从服务日志里的 open_id 填回白名单。

AI 润色

统一控制 AI 对文章的结构化增强。三项能力分别对应摘要与 frontmatter、正文补充块、以及全文正文润色。

连接与当前模型

先选择实际要使用的模型，再测试连通性。下面的 Provider 和模型池只负责配置与维护。

测试的是当前选择模型及其绑定的 Provider 连接配置。

Provider 管理

内置 provider 为只读协议定义，可编辑连接参数。也可以新增自定义 OpenAI Compatible provider。

这里只负责编辑连接配置，不决定实际运行使用哪个模型。

模型池

可以维护多个模型定义，但运行时只会使用顶部选中的一个。

导入 Clipper JSON 模板

直接导入 Obsidian Web Clipper 导出的 JSON 文件，自动映射为当前系统的 frontmatter、body 和解释器模板。

选择导出的模板 JSON 文件后会自动读取并填充，下方文本区可继续微调。

运行行为

控制单篇转换的隔离与成功后的清理策略。

关闭后会在服务端保留成功任务产物。失败任务始终会清理。

开启后每篇文章在独立子进程执行，单篇卡死不会拖垮整个服务。

单位为秒。超时后当前文章会标记失败，服务保持可用。
