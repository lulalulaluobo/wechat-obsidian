## 1. 系统概览

登录后，顶部导航栏提供以下页面：

| 页面 | 说明 |
|------|------|
| **转换中心** | 手动粘贴链接，转换单篇或批量文章为 Markdown |
| **同步来源** | 配置公众号后台凭证、搜索并添加公众号同步源、设置同步计划 |
| **文章库** | 管理已同步的文章索引，批量处理文章，设置执行计划 |
| **设置** | FNS 同步、图片模式、Telegram/飞书 Bot、AI 润色、行为参数等核心配置 |
| **退出登录** | 注销当前会话 |

## 2. 设置页面 - FNS 同步配置

> **路径**：设置 → FNS 同步

FNS（Fast Note Sync）是将转换好的 Markdown 文件同步到 Obsidian 的桥梁。这是必须首先配置的项目。

### 快捷导入方式

1. 在你的 Obsidian 中安装 Fast Note Sync 插件并配置好
2. 从 FNS 插件中复制配置 JSON（包含 `api`、`apiToken`、`vault` 字段）
3. 粘贴到 **FNS JSON Import** 文本框中
4. 点击 **解析并填充**，系统会自动填入下面的配置项

### 手动配置方式

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **FNS 服务地址** | FNS 服务器的地址 | `https://obsync.example.com` |
| **Vault 名称** | 你的 Obsidian 仓库名称 | `MyVault` |
| **目标目录** | 文件同步到仓库内的子目录 | `00_Inbox/微信公众号`（默认值） |
| **FNS Token** | FNS 服务的 API 认证令牌 | 填入后保存时自动加密存储 |

### 验证连接

点击 **检查 FNS 连接** 按钮，系统会测试连通性并验证 Vault 是否存在。状态栏显示连接结果。

> **提示**：页面刷新后 Token 会显示为掩码状态，这是正常的安全行为，不需要重新填写。

## 3. 设置页面 - 图片模式

> **路径**：设置 → 图片模式

控制文章中的图片如何处理。

### 微信热链模式（默认）

选择 **微信热链** 时，Markdown 文件中的图片直接使用微信原始 URL，不进行任何图片下载或上传操作。

- 优点：速度快，无需额外配置
- 缺点：依赖微信图片服务器的可用性，部分图片可能随时间失效

### S3 热链模式

选择 **S3 热链** 时，系统会将文章中的静态图片（jpg/png）上传到你指定的 S3 兼容存储，Markdown 中使用 S3 的公开访问 URL。GIF 和 SVG 仍保留微信原始链接。

选择 S3 模式后，需要填写以下配置：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **S3 Endpoint** | S3 兼容存储的 API 地址 | `https://s3.example.com` |
| **Region** | 存储区域 | `auto` |
| **Bucket** | 存储桶名称 | `my-images` |
| **Access Key ID** | 访问密钥 ID | — |
| **Secret Access Key** | 访问密钥（加密存储） | — |
| **路径模板** | 图片上传的路径格式 | `wechat/{year}/{filename}` |
| **公开访问 URL** | 图片的公开访问前缀 | `https://img.example.com` |

> **提示**：路径模板支持 `{year}`、`{filename}` 等占位符，可根据需要自定义目录结构。

## 4. 设置页面 - Telegram Bot

> **路径**：设置 → Telegram Bot

配置 Telegram Bot 后，你可以在 Telegram 中直接发送文章链接给 Bot，Bot 自动完成转换并同步到 Obsidian。

### 配置步骤

| 配置项 | 说明 |
|--------|------|
| **启用 Telegram Bot** | 勾选以启用此功能 |
| **转换完成后发送通知** | 勾选后，处理完成后 Bot 会回复转换结果（默认启用） |
| **Bot Token** | 从 [@BotFather](https://t.me/BotFather) 获取的 Bot API Token |
| **Webhook Secret** | 自定义的 Webhook 验证密钥，用于校验传入请求的合法性 |
| **Webhook 公开地址** | 本服务的外网可访问地址 | 
| **白名单 Chat ID** | 允许使用 Bot 的聊天 ID 列表（每行一个，或逗号分隔） |

### 操作流程

1. 在 Telegram 中通过 @BotFather 创建一个 Bot，获取 Bot Token
2. 获取你的 Chat ID（可通过 @userinfobot 查询）
3. 填写配置并保存，系统会自动向 Telegram 注册 Webhook
4. 向 Bot 发送微信公众号文章链接即可触发转换

> **注意**：Webhook 公开地址必须是 Telegram 服务器可以访问到的 HTTPS 地址。保存配置时系统会自动完成 Webhook 注册/注销。

## 5. 设置页面 - 飞书 Bot

> **路径**：设置 → 飞书 Bot

配置飞书自建应用 Bot 后，你可以在飞书中直接发送文章链接给 Bot，自动完成转换。

### 配置步骤

| 配置项 | 说明 |
|--------|------|
| **启用飞书 Bot** | 勾选以启用此功能 |
| **转换完成后发送通知** | 处理完成后是否回复结果（默认启用） |
| **App ID** | 飞书自建应用的 App ID（如 `cli_xxx`） |
| **App Secret** | 飞书应用的 Secret（加密存储） |
| **Verification Token** | 事件订阅的 URL 验证令牌 |
| **Encrypt Key** | 事件加解密密钥 |
| **Webhook 公开地址** | 本服务的外网可访问地址 |
| **白名单 open_id** | 允许使用 Bot 的用户 open_id 列表（每行一个） |

### 操作流程

1. 在 [飞书开放平台](https://open.feishu.cn/) 创建自建应用
2. 开启机器人能力，配置事件订阅
3. 将所需的凭证填入配置并保存
4. 在飞书中向 Bot 发送文章链接即可触发转换

> **注意**：目前仅支持私聊消息，不支持群聊。白名单在初次调试时可留空以便排查问题。

## 6. 设置页面 - AI 润色

> **路径**：设置 → AI 润色

AI 润色功能可为转换后的文章自动生成摘要、标签、Frontmatter 元数据，以及可选的正文润色。

### 总开关

| 开关 | 说明 |
|------|------|
| **启用 AI 摘要和 Frontmatter** | 生成结构化摘要、标签和 Frontmatter 元数据 |
| **启用正文补充块生成** | AI 额外生成一段补充说明（body_polish），插入到笔记中 |
| **启用全文润色** | AI 对文章正文进行重新排版和润色，生成更易读的 Markdown |

### 连接与模型

**当前模型**：下拉选择正在使用的 AI 模型。切换后立即生效。

**测试 AI 连通性**：点击按钮向当前选中的 AI 服务发送测试请求，验证配置是否正确，并显示延迟和响应预览。

### AI 提供商管理

系统内置了 5 个提供商预设：

| 提供商 | 类型 | 默认 Base URL |
|--------|------|---------------|
| OpenAI Compatible | openai_compatible | — |
| Anthropic | anthropic | `https://api.anthropic.com/v1` |
| Gemini | gemini | `https://generativelanguage.googleapis.com/v1beta` |
| Ollama | ollama | `http://127.0.0.1:11434` |
| OpenRouter | openrouter | `https://openrouter.ai/api/v1` |

每个提供商可配置：
- **显示名称**：自定义名称
- **Base URL**：API 地址（内置提供商可修改）
- **API Key**：认证密钥（加密存储）

你也可以点击 **添加自定义提供商** 来添加新的 OpenAI 兼容服务。自定义提供商可删除，内置预设不可删除。

### 模型池

可以配置多个模型定义，运行时只使用"当前模型"选中的那个。

每个模型需要配置：
- **显示名称**：界面中显示的标签
- **模型 ID**：发送给 API 的实际模型标识（如 `gpt-4o`、`claude-sonnet-4-20250514`）
- **绑定提供商**：关联到上述配置的某个提供商

### 提示词模板

| 模板 | 说明 |
|------|------|
| **解释器上下文模板** | 发送给 AI 的内容模板，默认为 `{{content}}`（即文章正文） |
| **内容润色提示词** | 全文润色功能启用时使用的 Prompt，指导 AI 重新排版文章 |
| **解释器提示词模板** | AI 处理的主要系统提示词，要求 AI 返回包含 `summary`、`tags`、`my_understand`、`body_polish` 的 JSON |
| **Frontmatter 模板** | 生成的 YAML Frontmatter 格式模板，支持 `{{title}}`、`{{author}}`、`{{summary}}`、`{{tags}}` 等占位符 |
| **正文模板** | 最终 Markdown 文件的正文格式模板，使用 Obsidian Callout 语法组织 AI 生成的内容 |

### Obsidian Web Clipper JSON 导入

如果你已有 Obsidian Web Clipper 的 JSON 模板文件，可以通过此功能一键导入：

1. 粘贴或上传 Clipper JSON 文件内容
2. 点击 **解析并填充**
3. 系统自动将 Clipper 模板映射为 Frontmatter 模板、正文模板、上下文模板和提示词模板

## 7. 设置页面 - 行为设置

> **路径**：设置 → 行为设置

控制转换任务的运行行为。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| **成功后自动清理临时目录** | 转换成功后删除临时工作目录。失败的任务文件会始终保留以便排查 | 启用 |
| **单次转换进程隔离** | 每篇文章的转换在独立子进程中运行，避免单个任务卡死影响整体服务 | 启用 |
| **单次转换硬超时（秒）** | 单篇转换的最大运行时间，超时后强制终止 | 180 秒 |

> **建议**：保持进程隔离和自动清理为启用状态。如果文章内容特别复杂或网络较慢，可适当增大超时时间。

## 9. 转换中心 - 手动转 Markdown

> **路径**：转换中心（首页）

### 单篇转换

1. 在输入框中粘贴微信公众号文章链接
2. 可展开 **高级选项** 进行以下设置：
   - **超时时间**：本次转换的超时秒数
   - **保存 HTML 调试文件**：保留中间 HTML 文件用于排查
   - **启用 AI 润色**：本次转换是否启用 AI 处理
3. 点击 **开始转换**

### 批量转换

1. 切换到 **批量转换** 标签
2. 在文本框中粘贴多个链接（每行一个），或上传包含链接的文件（支持 txt/md/csv 格式）
3. 配置高级选项（同上）
4. 点击 **开始批量转换**，系统创建后台批量任务

### 工作台状态栏

页面顶部显示当前系统状态：
- FNS 连接状态
- Vault 名称
- 当前图片模式
- AI 功能状态

## 10. 同步来源 - 公众号自动同步

> **路径**：同步来源

此页面用于配置微信公众号后台凭证，实现自动抓取公众号文章列表。

### 微信公众号后台凭证

三种配置方式：

**方式一：扫码登录（推荐）**
1. 点击 **扫码登录** 按钮
2. 使用微信扫描弹出的二维码
3. 系统自动获取并填充 Token 和 Cookie

**方式二：手动填写**
- 在 Token 和 Cookie 输入框中手动填入从浏览器中提取的凭证

**方式三：检查登录状态**
- 点击 **检查登录状态** 验证已保存的凭证是否仍然有效

> **注意**：微信公众号后台凭证有时效性，过期后需重新登录。

### 全局同步计划

配置自动同步所有公众号来源的计划：

| 配置项 | 说明 |
|--------|------|
| **频率** | `每天` 或 `每周` |
| **时间** | 具体的同步时间 |
| **星期** | 选择频率为"每周"时，指定哪一天 |
| **启用** | 勾选后按计划自动执行 |

> 同步计划仅抓取文章元数据（标题、链接等索引信息），不执行转换和 AI 润色。

### 搜索并添加公众号

1. 在搜索框中输入公众号名称关键词
2. 点击搜索
3. 在结果列表中找到目标公众号
4. 点击 **添加同步源**

### 管理同步源

已添加的公众号显示在列表中，每个来源提供以下操作：

| 操作 | 说明 |
|------|------|
| **首次/范围同步** | 指定日期范围，批量拉取该公众号在范围内的文章索引 |
| **增量同步** | 仅拉取上次同步后的新文章 |
| **删除** | 移除此同步源（已索引的文章不受影响） |

### 同步参数

在首次同步时，可设置 **开始日期** 和 **结束日期** 来控制拉取范围。

## 11. 文章库 - 批量管理与处理

> **路径**：文章库

文章库集中管理所有从公众号同步过来的文章索引。

### 筛选文章

页面顶部提供多种筛选条件：

| 筛选条件 | 说明 |
|----------|------|
| **公众号** | 按来源公众号筛选 |
| **处理状态** | 待处理 / 处理中 / 成功 / 失败 |
| **入库状态** | 已入库 / 未入库 |
| **执行记录** | 有执行记录 / 仅索引 |
| **发布日期范围** | 按文章发布时间筛选 |
| **跳过已入库文章** | 勾选后批量操作时排除已处理的文章（默认启用） |

### 全局执行计划

配置自动处理文章库中未执行文章的计划：

| 配置项 | 说明 |
|--------|------|
| **频率** | `每天` 或 `每周` |
| **时间** | 具体的执行时间 |
| **星期** | 选择频率为"每周"时，指定哪一天 |
| **启用** | 勾选后按计划自动执行 |

> 每次计划执行最多处理 20 篇文章，包含转换和可选的 AI 润色流程。

### 批量操作

1. 使用筛选条件缩小范围
2. 点击 **全选当前筛选结果**（或手动勾选）
3. 可进行的操作：
   - **批量加入处理队列**：将选中文章加入转换队列
   - **删除记录**：删除选中的文章索引记录

### 查看执行历史

点击文章行的 **查看执行记录** 按钮，可以看到该文章的所有转换尝试记录，包括状态、时间等信息。

## 典型工作流

以下是一个从零开始到日常使用的推荐流程：

```
第一步：基础配置
  └─ 设置 → FNS 同步 → 填写 FNS 连接信息并验证
  └─ 设置 → 图片模式 → 选择图片处理方式

第二步：按需启用扩展
  └─ 设置 → AI 润色 → 配置 AI 服务并测试连通性
  └─ 设置 → Telegram Bot 或 飞书 Bot → 配置即时转换通道

第三步：添加同步来源
  └─ 同步来源 → 扫码登录公众号后台
  └─ 搜索并添加关注的公众号
  └─ 执行首次范围同步

第四步：处理文章
  └─ 文章库 → 筛选待处理文章
  └─ 批量加入处理队列
  └─ 或配置全局执行计划实现自动化

第五步：日常使用
  └─ 通过 Telegram/飞书 Bot 发送链接即时转换
  └─ 或在转换中心手动粘贴链接
  └─ 文件自动同步到 Obsidian 指定目录
```
